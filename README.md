# 林一设计 · 个人网站

林一室内设计个人品牌网站，包含前台展示 + 管理后台的全栈应用。

---

## 在线地址

- **官网**: https://linyi.design
- **后台**: https://linyi.design/admin/

---

## 技术栈

| 层级 | 技术 |
|---|---|
| 前端 | 纯 HTML / CSS / JavaScript |
| 后端 | Node.js Serverless Functions（Vercel） |
| 数据库 | Supabase（PostgreSQL + Storage） |
| 部署平台 | Vercel + GitHub |

---

## 功能介绍

### 前台功能
- 响应式单页网站（首页 / 关于 / 作品 / 服务 / 联系）
- 作品展示区（多图轮播 + 灯箱翻看）
- 联系表单（提交到后台）
- 访客留言板（可开关）
- 全段落内容可编辑（通过管理后台）

### 管理后台
- 咨询留言管理（查看 + 删除）
- 访客留言管理
- 作品管理（列表 + 删除 + 上传 + 编辑 + 排序）
- 关于我（照片上传 + 文字编辑）
- 网站内容编辑（Hero / 理念 / 服务 / 联系 / 页脚）
- 系统设置（留言板开关）

## 项目开发历程

### 需求与解决方案记录

---

#### 1. 需求：删除作品排序功能，优化作品图片上传

**用户需求：**
- 删除后台上传作品页面的排序功能
- 在作品图片上传时看到图片预览，并支持拖拽排序
- 确保前端后端功能对应

**解决方案：**
- 移除前端 `admin/index.html` 中的排序输入框
- 实现图片预览功能，使用 `FileReader` 读取本地图片
- 实现拖拽排序功能，使用 HTML5 Drag & Drop API
- 后端移除数据库的 `sort_order` 排序字段
- 统一按创建时间降序显示作品

---

#### 2. 问题：本地后台密码问题

**用户需求：**
- 需要知道本地后台登录密码

**解决方案：**
- 查看环境变量配置，确认 `ADMIN_TOKEN` 为本地登录密码

---

#### 3. 问题：本地运行正常，上线后功能未更新

**用户需求：**
- 本地可以看到图片和拖拽排序，线上没有变化

**诊断与解决：**
- 问题根源：浏览器缓存或 CDN 缓存
- 解决方案：强制刷新页面（Ctrl + Shift + R）、清除浏览器缓存或等待 CDN 刷新
- 验证方法：通过查看 Vercel 部署日志确认代码是否已更新

---

#### 4. 问题：后台删除作品后，前台仍显示

**诊断与解决：**
1. 查看 Vercel 日志，发现 DELETE 请求出错
2. 分析原因：
   - DELETE 请求未调用 `getBody` 解析请求体，无法获取作品 ID
   - 数据库查询时列名错误（使用了不存在的列）
3. 修复：
   - 在 DELETE 请求中添加 `getBody` 调用
   - 修复数据库列名问题

---

#### 5. 需求：作品管理优化，解决多图上传后无法分辨的问题

**用户需求：**
- 上传作品多了无法分辨，希望一个作品生成一个文件夹
- 或者其他解决思路

**解决方案：**
- 分析后发现 Supabase Storage 文件夹操作较复杂
- 改用文件名命名规则：`{时间戳}_{序号}_{文件名}.{扩展名}`
- 这样既可以清晰分辨，又避免了文件夹操作的复杂性

---

#### 6. 问题：图片上传失败

**诊断与解决过程：**
1. 查看 Vercel 日志，定位到错误位置
2. 分析可能原因：
   - 代码逻辑重复混乱（存在重复上传逻辑）
   - 本地环境变量缺失
   - Supabase Storage 文件夹创建失败
   - 数据库表列名错误
3. 解决方案：
   - 移除重复的上传代码，优化上传流程
   - 指导用户创建 `.env.local` 文件配置环境变量
   - 移除文件夹创建逻辑，直接上传到根目录
   - 检查并修复数据库列名问题

---

#### 7. 问题：Supabase Storage 不允许创建文件夹

**诊断与解决：**
- 验证发现 Supabase Storage 实际上支持文件夹，但操作复杂
- 改为直接上传到根目录，使用带时间戳的文件名避免冲突

---

#### 8. 需求：实现编辑作品功能和排序功能

**用户需求：**
- 实现编辑作品功能
- 实现作品图片排序功能

**解决方案：**
- 前端：
  - 实现编辑模态框，加载现有作品信息和图片
  - 支持拖拽排序现有图片
  - 支持删除现有图片
  - 支持添加新图片
- 后端：
  - 实现 PUT 方法更新作品信息
  - 支持更新图片列表（删除/添加/排序）
  - 删除图片时同步删除 Storage 中的文件

---

#### 9. 需求：作品图片删除时同步删除 Storage 中的文件

**用户需求：**
- 在网站后台删除作品图片时，Storage 中的图片也同步删除
- 先调研是否可行，再实现

**调研与解决方案：**
1. 可行性分析：Supabase SDK 提供了删除文件的 API，完全可行
2. 实现方案：
   - 前端发送删除请求时，包含要删除的图片 URL 列表
   - 后端解析 URL，提取文件路径
   - 调用 Supabase Storage API 删除文件
   - 同时更新数据库中的图片列表
3. URL 解析策略：
   - 支持多种 URL 格式（Supabase 标准 URL、简化 URL、直接文件名）
   - 提取文件名并解码 URL 编码字符

---

#### 10. 问题：后台上传失败（最终关键问题）

**问题表现：**
- 上传作品时报"服务器错误"
- Vercel 日志显示 `Could not find the 'image_url'` 错误

**诊断与解决：**
1. 查看 Vercel 详细日志，定位到具体错误
2. 检查 Supabase 数据库表结构，发现实际列名是 `image_urls`（复数）
3. 问题根源：我之前错误地将代码中的列名改成了 `image_url`（单数）
4. 修复：
   - 将 `api/works.js` 中所有 `image_url` 改回 `image_urls`
   - 将 `api/admin.js` 中相关代码也修复
   - 测试验证上传、编辑、删除功能全部恢复正常

---

## 问题诊断方法总结

在开发过程中，遇到问题时使用以下方法诊断：

1. **查看日志**
   - Vercel Logs：查看部署后的运行日志
   - 浏览器控制台：查看前端错误
   - 网络请求：查看 API 请求和响应

2. **逐步调试**
   - 在关键位置添加 `console.log` 调试日志
   - 分步骤验证功能（先看参数是否正确，再看处理是否正确）

3. **检查数据库**
   - 登录 Supabase 后台查看表结构
   - 确认列名是否正确
   - 查看实际数据

4. **Git 历史回溯**
   - 使用 `git log` 查看提交历史
   - 使用 `git show` 查看历史版本代码
   - 对比修改前后的差异

5. **环境验证**
   - 确认环境变量配置正确
   - 验证数据库连接正常
   - 检查 Storage 权限

## 部署指南

### 前置条件
1. GitHub 仓库：https://github.com/W-wut/linyi
2. Vercel 账号（已绑定 GitHub）
3. Supabase 账号

### 环境变量
在 Vercel 项目 → **Settings** → **Environment Variables** 配置：

| 变量名 | 说明 |
|---|---|
| `SUPABASE_URL` | Supabase 项目 URL |
| `SUPABASE_SERVICE_KEY` | Supabase service_role 密钥 |
| `ADMIN_TOKEN` | 管理后台登录密码 |
| `GUESTBOOK_ENABLED` | 留言板开关（true/false） |

### 自动部署
代码推送到 GitHub 的 `main` 分支后，Vercel 会自动部署。
